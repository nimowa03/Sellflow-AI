## 1. 프로젝트 개요

### 1.1. 프로젝트 목표

**이커머스 셀러를 위한 리스크 검사 AI SaaS**

상품 등록 전, **상표권/저작권/정책 위반 표현**을 사전에 확인하여 이커머스 셀러가 법적 리스크를 줄이는 데 도움을 주는 **챗봇 기반 SaaS 플랫폼** 개발

### 1.2. 핵심 차별화 (vs 경쟁사)

| 항목 | 기존 서비스 (아이템스카우트, Windly 등) | **SellerBrain AI** |
|------|-----------------------------------|-------------------|
| 키워드 분석 | "검색량 높아요" (끝) | "검색량 높고, **상표권 확인까지**" ✅ |
| 상세페이지 | "만들어드려요" (법적 책임은 사용자) | "만들되, **정책 위반 표현 확인**" ✅ |
| 이미지 | "편집해드려요" (저작권은 사용자 책임) | "**AI 생성으로 저작권 리스크 감소**" ✅ |
| **UI 방식** | **복잡한 메뉴/대시보드** | **챗봇으로 대화하듯 사용** ✅ |
| **기능 사용** | **순차적 워크플로우** | **필요한 기능만 독립 사용** ✅ |

### 1.3. 타겟 사용자

**"리스크가 걱정되는 모든 이커머스 셀러"**

| 타겟 | 특징 | 페인 포인트 |
|-----|------|-----------|
| **이커머스 셀러** | 오픈마켓 판매자 | "이 키워드 괜찮을까?" |
| **1인 셀러** | 소호/개인 사업자 | "법적 문제 어떡하지?" |
| **위탁/구매대행** | 도매·해외 소싱 | "이 이미지 써도 되나?" |
| **부업 셀러** | 직장인·투잡 | "잘 모르고 했다가..." |
| **스타트업** | 초기 이커머스 팀 | "빠르게 검증하고 싶어" |

**지원 판매 유형** (종목 무관):
- ✅ 구매대행 (해외/국내)
- ✅ 위탁판매
- ✅ 자체 제작
- ✅ OEM/ODM
- ✅ 중고/리셀

## 2. WHY | 왜 만드는가

### 2.1. 시장의 문제점

```
┌─────────────────────────────────────────────────────────────────┐
│  기존 서비스들의 공통 문제:                                        │
│                                                                  │
│  "데이터는 주지만, 리스크 확인은 안 해준다"                        │
│  "자동화는 하지만, 법적 리스크는 사용자 몫"                        │
│                                                                  │
│  → 결과: 셀러들이 모르고 상표권 침해, 계정 정지 당함               │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2. 셀러들의 실제 페인 포인트

| 문제 | 빈도 | 결과 |
|------|------|------|
| 상표권 침해 키워드 사용 | 매우 높음 | 계정 정지, 손해배상 |
| 타사 이미지 무단 사용 | 높음 | 저작권 침해 고소 |
| 정책 위반 표현 사용 | 높음 | 상품 삭제, 벌금 |
| "최저가", "1위" 등 과장광고 | 보통 | 광고법 위반 |

### 2.3. SellerBrain AI의 접근 방식

```
┌─────────────────────────────────────────────────────────────────┐
│  SellerBrain AI가 도와드리는 것:                                  │
│                                                                  │
│  "업로드 전에 한번 확인해보세요.                                  │
│   문제될 키워드, 미리 알려드려요."                                │
│                                                                  │
│  🔍 상표권 확인: KIPRIS 50만+ 등록상표 DB 기반 사전 확인          │
│  📝 정책 위반 확인: 마켓별 정책 위반 표현 대조                    │
│  🖼️ 이미지 생성: AI 생성으로 저작권 리스크 감소                   │
│  💬 챗봇 UI: 대화하듯 편하게 사용                                 │
└─────────────────────────────────────────────────────────────────┘
```

> ⚠️ **면책 고지**: 본 서비스의 검사 결과는 참고용이며 법적 효력이 없습니다. 
> 정확한 상표권/저작권 확인은 KIPRIS, 변리사, 또는 법률 전문가 상담을 권장합니다.

### 2.4. 핵심 목표 (우선순위 순)

1. 🛡️ **리스크 사전 확인**: 상표권/저작권/정책 위반 표현 사전 파악
2. 💬 **챗봇 기반 UX**: 대화하듯 편하게 사용하는 인터페이스
3. 🎯 **독립 기능 사용**: 필요한 기능만 선택해서 사용
4. 🤖 **자동화 지원**: 상세페이지/이미지 생성 등 작업 효율화

### 3.2. 리스크 검사 시스템 (핵심 기능) 🛡️

| 검사 항목 | 데이터 소스 | 검사 방식 | 결과 |
|----------|-----------|----------|------|
| **상표권** | 특허청 KIPRIS (50만+) | RAG (ChromaDB) 검색 | ✅ 확인됨 / ⚠️ 주의 + 대체어 제안 |
| **정책 위반 표현** | 마켓별 정책 DB | 정규식 + AI 문맥 분석 | ✅ 확인됨 / ⚠️ 수정 제안 |
| **과장광고** | 공정위 가이드라인 | AI 분석 | ✅ 확인됨 / ⚠️ 표현 수정 |
| **저작권** | AI 생성 이미지 사용 | 원천 리스크 감소 | ✅ AI 생성 |

**검사 결과 예시** (챗봇 응답):
```
🔍 "노이즈캔슬링 에어팟 스타일 이어폰" 검사 결과:

✅ "노이즈캔슬링" - 확인됨 (일반 기술용어)
✅ "이어폰" - 확인됨 (일반 명사)
⚠️ "에어팟" - 주의! (Apple 등록상표 가능성)
   └── 대체어 제안: "이어버드", "TWS", "무선 이어폰"
⚠️ "스타일" - 주의 (모방 암시 가능성)
   └── 대체 표현: "타입", "형태"

💡 수정된 키워드: "노이즈캔슬링 TWS 타입 이어폰"

※ 본 결과는 참고용이며, 정확한 확인은 KIPRIS 또는 변리사 상담 권장
```
